這是一份為你的 **「GitHub 菸屁股自動捕獵機器人」** 量身打造的使用手冊。

`Score` (分數) 是這個系統的大腦核心，它代表了 **「這個機會的含金量與急迫性」** 。分數越高，代表多種訊號同時發生（便宜 + 人多 + 主力點火），你越應該優先關注。

以下是完整的系統運作邏輯與操作SOP：

-----
你的執行 SOP (湯姆熊永動機)
請把這個規則寫在你的投資筆記第一頁：

啟動條件： 當你的湯姆熊帳戶 全數結清變現 (例如 2026 聖誕節)。

結算盤點：

計算當下你的 全家族總淨值 (Total Net Worth)。

算出 新的 1% 金額 (例如 $12,000)。

多退少補：

若 湯姆熊現金 > 1%： 把多餘的錢轉走，拿去犒賞自己或存起來。

若 湯姆熊現金 < 1%： 從大金庫補錢進來，湊滿 1%。

Reset： 用新的 1% 額度，去尋找 2027、2028 年的新菸屁股。

結論： 這個方法非常科學且健康。 它讓你 「贏了守得住，輸了賠得起」。只要嚴格執行「提款」動作，不要讓湯姆熊帳戶無限膨脹變成 50%，你就是這個市場上最令人羨慕的 「長期贏家」。

### 一、 `Score` (分數) 的計算公式

這個分數不是隨機的，它是基於我們寫在 Python 裡的 `apply_rules` 邏輯累積而成的。**滿分為 5 分**。

| 得分來源 | 觸發條件 (Rule) | 背後邏輯 |
| :--- | :--- | :--- |
| **+1 分** | **基礎菸屁股** <br> (Ask \< $10 且 OI \> 5000) | **門票便宜且有人氣**。<br>這是最基本的入場券，代表這是一個合格的獵物。 |
| **+1 分** | **萬人塚 (Crowded)** <br> (OI \> 20,000) | **群眾共識極強**。<br>如 TSLA 960 那種 7 萬張單，會形成 Gamma 磁鐵，暴漲機率大增。 |
| **+3 分** | **主力點火 (Ignition)** <br> (成交量 \> 1000 且 比昨天暴增 2倍) | **現在就要動了！**<br>這是最權重的分數。代表沉睡的巨獸醒了，熱錢正在湧入。 |

#### 分數解讀：

  * **1\~2 分 (靜態埋伏)：** 適合掛低價等待 (Limit Order)。這通常是 `HOLD` 或普通菸屁股。
  * **3\~5 分 (動態追擊)：** 適合市價進場 (Market Order)。通常會標記為 `STRONG_BUY` 或 `BUY_WATCH`。

### 二、 訊號 (Action) 解讀與操作手冊

GitHub 每天早上會更新 `README.md`，你只需要看 **Action** 欄位來決定動作：

#### 1\. 🚀 STRONG\_BUY (強力買進)

  * **情境：** 同時滿足「便宜」+「人多」+「爆量點火」。
  * **意義：** 這就是我們要的 10x 起點。主力剛剛進場，且價格還在菸屁股區間。
  * **操作：** **不要掛太低，直接追價 (Market Buy) 或掛 Ask 價。** 速度是關鍵。

#### 2\. 👀 BUY\_WATCH (觀察/試單)

  * **情境：** 只有「爆量點火」，但可能不夠便宜，或是 OI 還沒累積到超大。
  * **意義：** 新的戰場正在形成。
  * **操作：**
      * 如果你很激進：先買 1 張卡位。
      * 如果你保守：加入券商的 Watchlist，設股價警示。

#### 3\. 🚬 HOLD (埋伏/觀望)

  * **情境：** 價格便宜，OI 巨大，但成交量死魚 (Vol 很低)。
  * **意義：** 這是目前清單上最多的狀態 (如 TSLA 960)。大部隊埋伏好了，在等訊號。
  * **操作：** **掛「芭樂價」 (Stink Bid)。**
      * 例如 Ask 6.65，你就掛 6.00 或 6.20。
      * 反正它不動，你就掛著等有緣人拋售給你。成交了就放著等點火，沒成交也沒差。

#### 4\. ⚠️ SELL\_ALERT (末日警示)

  * **情境：** 距離到期日 \< 45 天。
  * **意義：** 時間價值 (Theta) 正在加速吞噬你的本金。
  * **操作：** **強制出場**。不管賺賠，把剩下的錢拿回來，轉移到下一個 3\~6 個月後的合約。

### 三、 常見情境 (Scenarios) 與應對戰術

#### 情境 A：發現「沉睡巨獸」 (最常見)

  * **看到：** TSLA Jun 18 Strike 960，Score = 2 (菸屁股 + 萬人塚)，但 Volume 只有 200 張。
  * **心態：** 這是「存股」，不是當沖。
  * **動作：** 拿出你資金的 0.2%。掛一個稍微低於 Ask 的價格買進。買到後，設定一個 GTC (取消前有效) 的賣單在 +100% (翻倍) 的位置，然後忘記它。

#### 情境 B：發現「火箭發射」 (最稀有)

  * **看到：** 某天早上打開 GitHub，發現 **NVDA Strike 250** 突然變成 `🚀 STRONG_BUY`，Score 變成 5 分，成交量從昨天的 1000 變成 50000。
  * **心態：** 腎上腺素飆升，這就是暴擊時刻。
  * **動作：** 拿出資金的 0.5% \~ 1%。直接買進。進場後盯住 NVDA 的 5 日均線，只要不破線就一路抱到翻倍。

#### 情境 C：發現「主力落跑」

  * **看到：** 某個你持有的合約，OI 連續三天大幅下降 (例如從 5萬 掉到 3萬)。
  * **心態：** 派對結束了。
  * **動作：** 主力撤退了，你也該走了。即使沒虧很多，也要清倉，因為沒有 Gamma 磁鐵效應了。

### 四、 如何維護這套系統？

這個機器人是全自動的，但你作為「指揮官」，需要偶爾調整參數：

1.  **市場太熱，掃不到菸屁股？**

      * 如果股市大漲，所有 Call 都變貴，可能掃不到 `< $10` 的。
      * **調整：** 修改 `main.py` 裡的 `RULE_CONFIG['CHEAP_PRICE']`，改為 `15.0` 或 `20.0`。

2.  **想換口味，掃描別的妖股？**

      * **調整：** 修改 `main.py` 裡的 `TARGET_TICKERS`，加入 `SMCI`, `ARM` 或其他波動率高的股票。

3.  **如果 GitHub 沒更新？**

      * 去 GitHub Repo 的 "Actions" 分頁，看看是不是亮紅燈 (Failed)。如果是 403 Error (被擋)，可能需要休息幾天，或手動按一下 "Re-run jobs"。

### 總結你的每日 SOP
